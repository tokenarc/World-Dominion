# Instructions
1. **DRY Audit:** Identify repeated logic and move it into reusable utilities or hooks.
2. **Type Hardening:** Ensure every variable has a strict TypeScript definition; delete all 'any' types.
3. **Performance Scan:** Identify heavy React renders or inefficient loops and suggest O(n) optimizations.
4. **Pro Naming:** Rename vague variables into descriptive, professional identifiers.
